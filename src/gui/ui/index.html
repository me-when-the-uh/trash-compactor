    Compression settings
    These options decide how aggressively Trash Compactor searches for savings and how it behaves on slower drives.
    %
    Only files that can save at least this much space will be compressed. Higher values are more selective and faster; lower values squeeze out more space.
    Skips the strongest compression method, which can speed things up on systems where LZX would be too slow. Leave it enabled if you want the best balance of speed and savings.
    Limits the app to one worker at a time for older hard drives and noisy storage setups. On SSDs and modern drives, leaving it off usually gives better throughput.
    Chooses how file sizes are displayed in the interface. This changes the labels you see, but not the compression results themselves.

    Trash Compactor GUI

    Where we're going, we don't need backups!

    Report problems to: https://github.com/me-when-the-uh/trash-compactor.

    An intelligent graphical interface for Windows NTFS filesystem compression using entropy-based heuristics.

    For use with files and programs that rarely change - any file modifications will undo the compression for that file, so re-running this tool periodically is a good idea.

    Alternatively, the program can be used as a script to periodically run the compression on specific folders without calling the GUI window, which only appears if you launch the executable file without passing the directory that is to be compressed.

    Vibe-coded in Python with a webview-slopped GUI.
